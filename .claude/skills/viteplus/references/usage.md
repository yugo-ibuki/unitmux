# Vite+ 使い方リファレンス

Vite+は、Vite、Vitest、Oxlint、Oxfmt、Rolldown、tsdown、Vite Taskといったツールを統合し、ランタイム（Node.js）からパッケージマネージャー、フロントエンドツールチェーンまでを一本化して管理する**「Web開発のための統合ツールチェーン」**です。

## 1. インストールとプロジェクトの開始

グローバルコマンドラインツール `vp` をインストールします。

- **macOS / Linux:** `curl -fsSL https://viteplus.dev/install.sh | bash`
- **Windows:** `powershell -c "iwr https://viteplus.dev/install.ps1 -useb | iex"`

インストール後、以下のコマンドでプロジェクトを開始できます。

- **新規作成:** `vp create` を実行すると、対話形式で新しいモノレポ、アプリ、またはライブラリの雛形を作成できます。
- **既存プロジェクトの移行:** 既存のViteプロジェクトがある場合は、そのディレクトリで `vp migrate` を実行することで、Vite+の設定へ統合できます。
- **依存関係のインストール:** `vp install` を実行します。Vite+がプロジェクト内のロックファイルを自動検出し、適切なパッケージマネージャー（pnpm, npm, Yarn）を使用してインストールを行います。

## 2. 開発ワークフロー（Develop）

開発中によく使用するコマンドは以下の通りです。

- **開発サーバーの起動:** `vp dev` を実行すると、標準のVite開発サーバーが起動します。
- **静的チェック:** **`vp check`** は、コードの整形（Oxfmt）、リンター（Oxlint）、およびTypeScriptの型チェックを一括で実行する非常に高速なコマンドです。
- **単体テスト:** `vp test` でVitestによるテストを実行します。ウォッチモードにする場合は `vp test watch` を使用します。
- **個別ツール:** 必要に応じて `vp lint`（リンターのみ）や `vp fmt`（フォーマッターのみ）を個別に叩くことも可能です。

## 3. ビルドとプレビュー（Build）

プロジェクトを本番用に書き出す際のコマンドです。

- **アプリのビルド:** `vp build` は、Vite 8とRolldownを使用して本番用のアセットを生成します。
- **ライブラリのビルド:** `vp pack` は、tsdownを使用してライブラリやスタンドアロンの実行ファイルをビルドします。
- **プレビュー:** `vp preview` を実行すると、ビルドされた成果物をローカルで確認できます。

## 4. タスクの実行と依存関係の管理

- **スクリプトの実行:** `vp run <名前>` を使うことで、`package.json` に書かれたスクリプトや、`vite.config.ts` で定義したカスタムタスクを実行できます。
- **パッケージの追加・削除:** `vp add <パッケージ名>` や `vp remove <パッケージ名>` を使用します。これにより、背後にあるパッケージマネージャーを意識せずに一貫した操作が可能です。
- **バイナリの実行:** ローカルまたはリモートのバイナリを実行するには `vpx <コマンド>` を、プロジェクト内の `node_modules/.bin` にあるものを実行するには `vp exec <コマンド>` を使用します。

## 5. 環境管理とメンテナンス

- **Node.jsの管理:** `vp env` コマンドでNode.jsのバージョンを管理できます。`vp env pin` でプロジェクトごとにバージョンを固定することも可能です。
- **Gitフックの設定:** `vp config` を実行すると、コミット時に自動でチェックを走らせるためのGitフック（コミットフック）をインストールできます。
- **アップデート:** Vite+自体を更新するには `vp upgrade` を使用します。

## 設定の集約

Vite+の大きな特徴は、**設定を `vite.config.ts` に集約できる点**です。リンター（`lint` ブロック）、フォーマッター（`fmt` ブロック）、テスト（`test` ブロック）、ライブラリビルド（`pack` ブロック）などの設定をすべて1つのファイルに記述するため、多数の設定ファイル（`.eslintrc`, `.prettierrc` など）を管理する手間が省けます。
