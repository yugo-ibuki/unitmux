# Vite+ コマンド一覧

Vite+（`vp`）の全コマンドリファレンスです。プロジェクトの開始から開発、テスト、ビルド、メンテナンスまで、Web開発のライフサイクル全体をカバーします。

## 1. プロジェクトの開始と設定 (Start)

新しいプロジェクトの作成や既存環境の移行、初期設定に関わるコマンドです。

- **`vp create`**: 新しいモノレポ、アプリケーション、またはライブラリの雛形を対話形式で作成します。組み込みテンプレートのほか、リモートのGitHubリポジトリなどをソースにすることも可能です。
- **`vp migrate`**: 既存のVite、Vitest、ESLint、Prettierなどを使用しているプロジェクトをVite+環境へ移行します。設定を `vite.config.ts` に集約し、スクリプトをVite+形式に書き換えます。
- **`vp config`**: Gitコミットフックのインストールやエージェント統合の設定を現在のプロジェクトに対して行います。デフォルトでは `.vite-hooks` ディレクトリにフックを書き込みます。
- **`vp staged`**: Gitでステージング（`git add`）されたファイルに対してのみ、`vite.config.ts` の `staged` ブロックで定義されたチェックを実行します。
- **`vp install`**: プロジェクトの依存関係をインストールします。`package.json` やロックファイルを自動検出し、pnpm、npm、Yarnの中から適切なパッケージマネージャーを自動で使い分けます。
- **`vp env`**: Node.jsの実行環境を管理します。
  - `vp env setup / on / off`: 管理モードの有効化・無効化を切り替えます。
  - `vp env default / pin / use`: グローバル、またはディレクトリ（プロジェクト）単位で使用するNode.jsのバージョンを固定・指定します。
  - `vp env install / list`: Node.jsのインストールやバージョン確認を行います。

## 2. 開発と品質管理 (Develop)

日々のコーディングやコードの品質維持に使用するコマンドです。

- **`vp dev`**: Vite開発サーバーを起動します。標準的なViteの体験をそのまま提供します。
- **`vp check`**: **コード整形（Oxfmt）、リンティング（Oxlint）、型チェック（tsgolint）を一度に実行**する最も推奨される静的チェックコマンドです。個別に実行するよりも大幅に高速です。
- **`vp lint`**: Oxlintを使用してコードのリンティングを行います。JavaScriptベースのESLintプラグインも限定的にサポートしています。
- **`vp fmt`**: Oxfmt（Prettier互換の高速フォーマッター）を使用してコードを整形します。
- **`vp test`**: Vitestを使用してテストを実行します。
  - 通常の `vp test` は1回のみ実行されます。継続的にテストを走らせる場合は **`vp test watch`** を使用します。

## 3. タスクの実行とキャッシュ (Execute)

スクリプトやバイナリの実行を効率化するコマンドです。

- **`vp run <task>`**: `package.json` のスクリプトや `vite.config.ts` で定義したカスタムタスクを実行します。
  - `vite.config.ts` で定義されたタスクは**デフォルトでキャッシュが有効**になり、2回目以降の実行が高速化されます。
  - `-r`（再帰的実行）や `--filter`（パッケージ指定）など、モノレポでの実行に便利なフラグをサポートしています。
- **`vp cache clean`**: `vp run` によって生成されたタスクのキャッシュを削除します。
- **`vpx <command>`**: ローカルまたはリモートのnpmパッケージにあるバイナリを実行します。`npx` のVite+版です。
- **`vp exec <command>`**: プロジェクト内の `node_modules/.bin` にインストールされているバイナリを直接実行します。
- **`vp dlx <package>`**: パッケージをプロジェクトの依存関係に追加することなく、一時的にダウンロードしてバイナリを実行します。

## 4. ビルドとプレビュー (Build)

本番環境向けの成果物を作成するためのコマンドです。

- **`vp build`**: 本番用のWebアプリケーションをビルドします。内部的にはVite 8とRolldownを使用しています。
- **`vp pack`**: ライブラリや、Node.jsなしで動作する**スタンドアロンの実行ファイル（.exe等）**をビルドします。これには `tsdown` が使用されます。
- **`vp preview`**: `vp build` で生成された本番用ビルドをローカルでプレビューします。

## 5. 依存関係の管理 (Manage Dependencies)

パッケージマネージャーの操作を抽象化し、一貫したインターフェースを提供します。

- **`vp add` / `vp remove`**: パッケージの追加と削除を行います。
- **`vp update`**: 依存パッケージを最新バージョンに更新します。
- **`vp dedupe`**: 重複した依存関係を整理してロックファイルを最適化します。
- **`vp outdated`**: 更新が可能なパッケージの一覧を表示します。
- **`vp why <pkg>`**: 特定のパッケージがなぜプロジェクトに含まれているのか（依存グラフ）を表示します。
- **`vp info <pkg>`**: レジストリにあるパッケージのメタデータを表示します。
- **`vp link` / `vp unlink`**: ローカルパッケージのシンボリックリンクを管理します。
- **`vp pm <command>`**: 基盤となるパッケージマネージャー（pnpm等）の固有コマンドを直接実行します。

## 6. メンテナンス (Maintain)

Vite+自体の管理を行うコマンドです。

- **`vp upgrade`**: グローバルにインストールされている `vp` バイナリ自体を最新バージョンに更新します。
- **`vp implode`**: **Vite+をマシンから完全に削除**します。これには、管理されているNode.jsランタイムや設定データも含まれます。
