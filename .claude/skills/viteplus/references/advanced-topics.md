# Vite+ 高度なトピック

基本的な使い方やコマンド一覧を超えた、より深い活用のためのガイドです。

## 1. 統合設定ファイル (`vite.config.ts`) 構成ガイド

Vite+の最大の特徴は、あらゆるツールの設定を一つのファイルに集約できる点です。

- **内容:** `lint`, `fmt`, `test`, `pack`, `staged`, `run` といった各ブロックがどのように相互作用し、どのように記述すべきかのベストプラクティス。
- **理由:** 従来の `.eslintrc` や `prettierrc` からの移行先として、このファイルの構造を深く理解することが保守性の向上に直結するためです。

## 2. モノレポ・ワークスペース管理ガイド

複数のパッケージを管理する大規模プロジェクト向けの運用ドキュメントです。

- **内容:** `vp run -r`（再帰実行）や `--filter` を活用した特定パッケージへの操作、および `package.json` の依存関係に基づいたタスク実行順序の制御方法。
- **理由:** Vite+はワークスペースを意識した設計になっており、効率的なビルドやテストのパイプラインを組むために不可欠な知識だからです。

## 3. 高度なタスクキャッシュとカスタムタスク設計

標準コマンド以外の独自のワークフローを高速化するためのガイドです。

- **内容:** `dependsOn` を用いたタスク間の依存関係の定義、`input` オプションによるキャッシュ無効化条件の最適化、および環境変数（`env` / `untrackedEnv`）がキャッシュに与える影響の詳細。
- **理由:** 自動ファイル追跡機能を正しく理解し、不要なキャッシュミスを防いで開発速度を最大化するためです。

## 4. ライブラリおよびスタンドアロン実行ファイルの作成 (Pack)

アプリケーション開発ではなく、配布可能なパッケージを作るためのドキュメントです。

- **内容:** `tsdown` を使用した型定義ファイル（dts）の生成、複数出力形式の設定、および Node.js なしで動作するスタンドアロン実行ファイル（.exe等）のビルド方法。
- **理由:** アプリ開発の `vp build` とは異なる、ライブラリ開発特有のワークフローをカバーするためです。

## 5. AI コーディングエージェント統合ガイド

AIツールとの連携を強化するためのモダンな開発ガイドです。

- **内容:** `--agent` フラグを使用したエージェント指示ファイルの生成、およびエージェントに移行作業を依頼するためのプロンプトの活用方法。
- **理由:** Vite+はエージェントとの親和性を重視して設計されており、これを活用することでプロジェクトのセットアップや移行を劇的に効率化できるためです。

## 6. 本番環境および CI/CD への導入ガイド

ローカル開発環境から本番デプロイまでの自動化に特化したドキュメントです。

- **内容:** GitHub Actions における `voidzero-dev/setup-vp` の活用、依存関係キャッシュの自動化、および CI 上での静的チェック（`vp check`）の最適化。
- **理由:** 個別のコマンド実行だけでなく、開発パイプライン全体に Vite+ を組み込むことで、チーム全体のコード品質を維持するためです。

## 7. トラブルシューティングと診断の詳細

問題が発生した際の自己解決力を高めるためのリファレンスです。

- **内容:** 型チェックがスキップされる条件（`baseUrl` 設定の影響など）、キャッシュがヒットしない理由の調査方法、`vp env doctor` を使った環境診断。
- **理由:** アルファ版である現在のステータスにおいて、開発者が自身の環境特有の問題を特定しやすくするためです。
